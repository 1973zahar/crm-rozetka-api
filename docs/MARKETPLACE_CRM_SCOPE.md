# Marketplace + B2B CRM scope

Ціль: CRM фокусується на операціях маркетплейсів і B2B контурі. B2B заявки, відповідальне зберігання, звіти продажу клієнтом і кабінети клієнтів є першими екранами цієї версії поряд із marketplace-операціями.

## Основні модулі

1. `Панель` — всі зміни у статусах і змінах на екрані.
   - `Воронки` — етапи руху маркетплейс-замовлень.
2. `Продажі`.
   - `Замовлення`.
   - `Логістика з рухом замовлень`.
   - `Комунікація з покупцями по замовленнях`.
3. `B2B`.
   - `Огляд`.
   - `Заявки клієнтів`.
   - `Відвантаження і продажі`.
   - `Відповідальне зберігання`.
   - `Кабінети клієнтів`.
4. `Товари`.
   - `Каталог`.
   - `Залишки на складі`.
   - `Публікації`.
5. `Аналітика та звіти`.
6. `Фінанси`.
   - `Оплати за замовлення`.
   - `Розрахунки із маркетплейсами`.
7. `Налаштування`.
   - `Маркетплейси`.
   - `Логістика`.
   - `Комунікатори та зв'язок`.
   - `Воронки`.
   - `Команда`.

## Rozetka API inbound-only

CRM завантажує реальні дані з Rozetka у CRM і не відправляє зміни назад у Rozetka.

Потрібно заповнити `.env` у папці `marketplace-crm`:

```ini
ROZETKA_API_TOKEN=ваш_токен
```

або:

```ini
ROZETKA_USERNAME=ваш_логін
ROZETKA_PASSWORD=ваш_пароль
```

Доступні завантаження:

- `GET /api/rozetka/items/search/all` — активний каталог Rozetka → `Товари / Каталог` і `Товари / Публікації`.
- `GET /api/rozetka/goods/new/all` — нові або неопрацьовані товари Rozetka → CRM.
- `GET /api/rozetka/orders/import` — замовлення Rozetka → `Продажі / Замовлення`.
- `GET /api/rozetka/image?url=...` — проксі фото Rozetka для карток товарів.

Повний сирий JSON зберігається у `product.rozetka.raw` або `order.rozetka.raw`.

## Перший робочий процес

1. Імпортувати товари з Rozetka або іншого каналу.
2. Створити або оновити внутрішній товар і публікацію маркетплейсу.
3. Імпортувати замовлення.
4. Перевірити покупця, товар, оплату, доставку й статус.
5. Передати замовлення в комплектацію або доставку.
6. Підтягнути ТТН та оплату через API.
7. Бачити результат у панелі й аналітиці.

## Поза першим фокусом

- повний фінансовий блок;
- прихідні документи як самостійний модуль;
- серійний облік як окремий операційний екран;
- BAS/BAF як головний сценарій.

Ці частини можуть лишатися всередині коду як підтримуюча логіка або майбутні інтеграції, але головний інтерфейс веде користувача через marketplace і B2B операції.
